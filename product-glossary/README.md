# Product glossary

When writing for Meridian, avoid buzzwords, slang, and technical jargon. A list of product-specific words is outlined below.

## A

### Aura

Aura is an AI assistant included within the Meridian dashboard.&#x20;

* **Terms to avoid**: chatbot, agent, Meridian Aura (just Aura)
* **See also**: [Meridian product names](product-names.md)

## B

### Budget

Used to estimate payment terms. _Budget_ is a more user-friendly term than payment terms. It also reminds the clients that money is a vital part of the project and partnership.

* **Terms to avoid**: payment terms

## C

### Client

Used to describe the person paying for the project work.&#x20;

* **Terms to avoid**: business owner

### Freelancer

Used to describe the person who performs the work. Freelancer feels like a more generalized term instead of contractor or consultant.

* **Terms to avoid**: contractor, consultant

### Hours

Used to track the amount of effort that has been completed on the project. Instead of saying 'effort,' most users would understand that 'hours' meant time spent.

* **Terms to avoid**: effort, time

## L

### Log in

Used to describe when a user logs into the app. We don’t use sign in because it sounds and looks too similar to sign up.

* **Terms to avoid**: sign in

## P

### Payment

Used to describe money sent for the project.  Payment is a generalized enough term that works for both team members and  client and is commonly understood.

* **Terms to avoid**: invoice, bill, money

### Project

Used to describe the work being proposed/completed. Project is a term that could cover all kinds of work.

* **Terms to avoid**: work, job

## S

### Scope

Used to provide the description of work to be completed on the project. Scope is a term commonly used in project management, and description could be too vague.

* **Terms to avoid**: description

### Sign up

Used to describe when a user creates a new account.

* **Terms to avoid**: create account

## T

### Track

Used to describe how a freelancer will update a client on the work completed. This is a common term that most users understand.

* **Terms to avoid**: create account
